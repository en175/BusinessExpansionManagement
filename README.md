# 广州仲裁委员会 · 业务拓展管理系统

> 基于广州仲裁委员会现有案件管理系统设计风格开发的「业务拓展管理」模块 Demo，供甲方预览确认交互方案。

## 功能模块

| 模块 | 功能说明 | 可访问角色 |
|------|---------|-----------|
| 登记活动 | 填写业务拓展活动信息，提交审批 | 全员 |
| 我的记录 | 查看本人提交的记录及审批状态 | 全员 |
| 待我审批 | 审批本部门提交的活动记录 | 部门正职 |
| 查询与统计 | 多维度检索全委活动记录，导出 Excel | 部门正职 / 战略发展部指定人员 |

## 角色权限

| 角色 | 示例用户 | 权限范围 |
|------|---------|---------|
| 普通员工 | 鲍淑婷（战略发展部） | 登记 + 我的记录 |
| 部门正职 | 余莨桢（战略发展部部长） | 登记 + 我的记录 + 待我审批 + 查询统计 |
| 战略发展部专员 | 许越（战略发展部） | 登记 + 我的记录 + 查询统计 |

> 右上角角色切换器可一键演示不同角色的页面权限与数据视图。

## 审批流程

```
员工填报
   ↓
推送至部门正职审批
   ├─ 填报人 = 部门正职 → 自动通过，系统备注「自动通过」
   └─ 部门无正职（如案件管理三部）→ 推副职审批
         ↓
      通过 → 自动抄送战略发展部正职及指定人员
      退回 → 通知填报人修改，可撤回重提
```

## 登记表单字段

- **基本信息**：活动标题、填报人（自动）、我委对接人、牵头部门、配合部门
- **活动信息**：活动时间（时间段）、活动形式（走访 / 来访 / 调研 / 合作洽谈 / 研讨会 / 宣讲会 / 其他）、活动规模（人数）
- **参与人员**：我委参与人员（支持姓名实时搜索）、对方参与人员（姓名+职务，可动态添加多条）
- **开拓对象**：机构名称、纳税人识别号（预留与案件关联）、所属行业、所在地区（省市联动），支持添加多个对象
- **活动详情**：活动目的及成效、反馈意见建议、后续需跟进事项
- **佐证材料**：附件上传（非必填）、备注

## 本地开发

```bash
# 安装依赖
npm install

# 启动开发服务器（http://localhost:3000）
npm run dev

# 构建生产包
npm run build
```

## 部署（Netlify）

项目已配置 `netlify.toml`，推送到 GitHub 后 Netlify 自动触发 CI/CD：

```toml
[build]
  command = "npm run build"
  publish = "dist"
```

## Mock 数据说明

- 人员数据来源：广州仲裁委员会人员调整一览表（16 个部门，196 人）
- 预置 7 条活动记录，涵盖「已通过」「待审批」「已退回」「自动通过」各状态
- 战略发展部正职（余莨桢）有 2 条自己提交的记录（自动通过）
- 部门正职角色下，「待我审批」有 2 条来自战略发展部员工的待审批记录
